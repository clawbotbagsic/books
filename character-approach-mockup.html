How should parents choose a character?

Three directions — each changes the UX and generation pipeline differently. Click to select.

A

Pre-made Creatures — Pick & Go Recommended

8 lovable original characters (Stitch-style creatures) shown as a visual grid. Parent taps one, done. No customization. Each has a pre-generated FLUX anchor stored permanently — Phase 1 (anchor generation) is skipped entirely on every book.

🛸 Zib the alien 🐉 Ember the dragon 🦕 Pip the dino ☁️ Boo the cloud 🤖 Cog the robot 🦊 Fern the fox 🐻 Mochi the bear ✨ Lumi the jellyfish

Pros

  • Fastest UX — one tap
  • Saves $0.025 anchor cost per book
  • Characters always look great
  • Consistent across all books
  • Simplest to build

Cons

  • No visual link to the child
  • Fixed roster (can expand later)
B

Dicebear Parametric Builder

Parent picks skin tone, hair style, and hair color using a simple visual picker. A Dicebear SVG previews instantly in the form. Those params translate into a FLUX Dev prompt → real anchor generated once on submit (~30s wait before book starts).

Pros

  • Child representation built in
  • Live Dicebear SVG preview
  • Params → FLUX prompt is clean

  • Age rendering problem may return
  • More form steps = lower completion
  • Dicebear ≠ book art style
C

Pick a Creature + Customize Color

Parent picks one of the 8 creature types, then picks a color variant (e.g. "Ember in purple" vs "Ember in red"). We pre-generate 2–3 color variants per creature. Still no anchor generation at runtime — all anchors pre-made.

🐉 Ember → 🔴 Red 🔵 Blue 🟣 Purple ✓

Pros

  • Fun, playful UX
  • Some personalization
  • Still no runtime generation cost

Cons

  • 8 creatures × 3 colors = 24 anchors to pre-generate
  • More complex form UI
  • Scope creep risk vs. A

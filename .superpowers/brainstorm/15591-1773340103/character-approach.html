How should parents choose their character?

Three directions — each changes the UX and the generation pipeline differently

A

Pre-made Creatures — Pick & Go

8 lovable original characters (Stitch-style creatures) shown as a grid. Parent taps one, done. No customization. Each character has a pre-generated FLUX anchor stored permanently — Phase 1 is skipped entirely.

🛸 Zib the alien 🐉 Ember the dragon 🦕 Pip the dino ☁️ Boo the cloud 🤖 Cog the robot 🦊 Fern the fox 🐻 Mochi the bear ✨ Lumi the jellyfish

Pros

  • Fastest UX (one tap)
  • No generation cost for anchor
  • Characters always look great
  • Consistent across all books

Cons

  • No personalization to the child
  • Limited to 8 options
B

Dicebear Parametric Builder

Parent picks attributes (skin tone, hair style, hair color, eye color) using a simple visual picker. A Dicebear SVG previews instantly. Those params feed into a FLUX Dev prompt → real anchor generated once on submit.

Skin Tone
Hair

Pros

  • Child representation built in
  • Live SVG preview via Dicebear
  • Params → FLUX prompt is clean

Cons

  • Still generates anchor on submit (~30s wait)
  • Age problem may resurface
  • More form steps
C

Pick a Creature + Customize It

Parent picks one of the 8 creature types, then optionally tweaks 2–3 attributes (color, accessory, style). Customizations adjust the stored anchor slightly via FLUX img2img — or just vary the prompt seed. Dicebear previews the creature style while they choose.

🐉 Ember → 🔴 Red 🔵 Blue 🟣 Purple ✓

Pros

  • Fun, playful UX
  • Some personalization
  • Pre-made anchors still possible

Cons

  • Most complex to build
  • Many anchor variants to pre-generate
  • Scope creep risk
